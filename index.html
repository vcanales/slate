    Introduction

    Welcome to the Port API definition.

    API endpoint

    All requests should be directed at our API server, located at

    https://api.port.cl/v1

    Authentication

    Port uses API keys to allow access to the API, which should be included in the header of every authenticated request:

    Authorization: Bearer <api_key>

    You must replace with your API key.

    Pagination

    All available resources support requesting a List object which delivers a paginable result.

    {
  "object": "list",
  "has_more": false,
  "url": "/v1/communities",
  "data": []
}

    Query Parameters

    Parameter Default Description
    limit 10 Sets the amount of records to be included in the call.
    offset 0 Start from the specified consecutive element

    List Response Format

    Attribute Description
    object String representing the object's type. value: list
    has_more Whether or not there are more records after this set
    total_count Total amount of records
    data Array containing the actual response objects

    Communities

    The Community object

    {
  "object": "community",
  "id": "com_1",
  "name": "Comunidad Edificio Chile España",
  "address": "Alonso de Ercilla 3055, Ñuñoa, Santiago",
  "location": "1231,14234,23424,4324",
  "admin": null,
  "divisions": {
    "object": "list",
    "has_more": false,
    "url": "/v1/communities/1/divisions",
    "total_count": 0,
    "data": [

    ]
  }
}
    Attribute Description
    object String representing the object's type. value: community
    id Community's unique identifier
    name Name of the community
    address String representing the physical address of the Community
    location Exact coordinates of the Community
    admin ID of the user in charge
    divisions Array of division objects, representing the internal, organizational subdivisions of a community
    example: 'Torre A' and 'Torre B' might be subdivisions

    Errors

    This error section is stored in a separate file in `includes/_errors.md`. Slate allows you to optionally separate out your docs into many files...just save them to the `includes` folder and add them to the top of your `index.md`'s frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
